创建 crates/github_adapter/src/releases.rs：

1. 定义 `ReleaseInfo` struct: release_id (i64), tag_name (String), name (Option<String>), body (Option<String>), prerelease (bool), published_at (Option<String>), html_url (String)
2. 定义 `TagInfo` struct: name (String), sha (String), commit_url (Option<String>)
3. `fetch_latest_releases(owner, name, since_cursor: Option<String>, token) -> Result<Vec<ReleaseInfo>>`:
   - 创建 octocrab client（复用 auth.rs 模式: Octocrab::builder().personal_token(token).build()）
   - 调用 octocrab.repos(owner, name).releases().per_page(30).send()
   - 如果 since_cursor 存在，过滤掉 published_at <= since_cursor 的项
   - 返回 Vec<ReleaseInfo>
4. `fetch_latest_tags(owner, name, since_tag: Option<String>, token) -> Result<Vec<TagInfo>>`:
   - 调用 octocrab.repos(owner, name).list_tags().per_page(30).send()
   - 如果 since_tag 存在，只返回 since_tag 之后的 tags（按名称排序时比较）
   - 返回 Vec<TagInfo>

错误处理使用 thiserror（遵循 auth.rs 模式）。使用 octocrab 已有 models。

在 lib.rs 中添加 `pub mod releases;`。
