创建 crates/application/src/subscription.rs，遵循 topk.rs 的编排模式：

1. `subscribe(conn, request: CreateSubscriptionRequest, token) -> Result<SubscriptionDto>`:
   - 检查是否已有活跃订阅 (get_subscription_by_repo_id)
   - 创建 Subscription 对象
   - 调用 create_subscription 存入 DB
   - 调用 fetch_repo_info 获取仓库信息（用于首次 baseline）
   - 设置 cursor_release_id = 当前最新 release ID, cursor_tag_name = 当前最新 tag name
   - 调用 update_subscription_cursors 保存 baseline
   - 返回 SubscriptionDto

2. `unsubscribe(conn, subscription_id) -> Result<()>`:
   - 调用 update_subscription_state(id, Archived)

3. `pause_subscription(conn, subscription_id) -> Result<()>`:
   - 获取当前状态，若 Active 则转 Paused，若 Paused 则转 Active

4. `list_subscriptions(conn) -> Result<Vec<SubscriptionDto>>`:
   - 调用 subscription_repository::list_subscriptions
   - 转换为 SubscriptionDto 列表

5. `sync_subscriptions(conn, token) -> Result<usize>`:
   - 获取所有 active subscriptions
   - 遍历每个 subscription:
     - try: fetch_latest_releases(owner, name, cursor_release_id, token)
     - try: fetch_latest_tags(owner, name, cursor_tag_name, token)
     - 对每个新 release: 创建 Signal::new_release(), 调用 insert_signal (幂等)
     - 对每个新 tag: 创建 Signal::new_tag(), 调用 insert_signal (幂等)
     - 更新 subscription cursors + last_successful_sync_at
     - catch: log error, continue to next subscription
   - 返回成功同步的订阅数量

在 lib.rs 中添加 `pub mod subscription;`。
