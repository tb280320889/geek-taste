# Plan 01-03: 导航壳与路由

**Phase:** 1 — 项目脚手架与认证
**Requirements:** FOUND-04
**Status:** Ready

## Goal

搭建应用的导航骨架：左侧 Sidebar、5 个页面路由、认证状态守卫。用户能看到完整的导航结构并在页面间切换。

## Scope

### apps/desktop-ui/src/routes/
1. `+layout.svelte` — 主布局
   - 左侧 Sidebar 组件
   - 右侧内容区（<slot>）
   - 认证状态检查：未认证时显示内容区提示卡
   - 全局样式：暗色主题优先（参考 Linear/Vercel）

2. `+page.svelte` — Home 页面（占位 + 欢迎信息）
3. `topk/+page.svelte` — TopK 页面（占位）
4. `subscriptions/+page.svelte` — 订阅页面（占位）
5. `resources/+page.svelte` — 资源雷达页面（占位）
6. `rules/+page.svelte` — 规则页面（占位）
7. `settings/+page.svelte` — 设置页面（占位，Plan 05 详细实现）
8. `onboarding/+page.svelte` — 认证页面（占位，Plan 04 详细实现）

### apps/desktop-ui/src/lib/
1. `components/Sidebar.svelte` — 侧边栏组件
   - 5 个一级导航项：Home, TopK, Subscriptions, Resources, Rules
   - 底部齿轮图标 → Settings
   - 当前页面高亮
   - 折叠/展开功能（v1 可选）
   - 图标：使用简单的 SVG 内联图标（避免额外依赖）

2. `stores/auth.ts` — 认证状态 store
   - Svelte writable store: `authStatus` (Authenticated | Unauthenticated | Loading)
   - `currentUser` writable store (UserDto | null)
   - `initAuth()`: 调用 Tauri IPC get_current_user
   - `logout()`: 调用 remove_github_token → 重置 stores

3. `stores/settings.ts` — 设置状态 store
   - Svelte writable store: `settings` (SettingsDto)
   - `loadSettings()`: 调用 Tauri IPC get_settings
   - `updateSettings()`: 调用 Tauri IPC update_settings

4. `ipc/tauri.ts` — Tauri IPC 调用封装
   - 封装所有 invoke() 调用为 typed functions
   - 导出: validateToken, storeToken, loadToken, removeToken, getCurrentUser, getSettings, updateSettings, fetchRepoInfo

### apps/desktop-ui/src/
1. `app.css` — 全局样式
   - CSS 变量：颜色、间距、字体
   - 暗色主题变量
   - Sidebar 和布局样式

### Key Design Decisions
- 使用 SvelteKit 文件路由（已配置 SSR 禁用）
- Sidebar 使用 CSS Grid 布局（sidebar 240px + content 1fr）
- 认证状态通过 Svelte store 管理，layout 订阅变化
- 未认证时：导航壳可见 + 内容区显示"请先连接 GitHub"提示卡
- 图标使用内联 SVG（v1 不引入 icon 库）
