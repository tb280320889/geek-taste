# Plan 01-05: Settings 页面

**Phase:** 1 — 项目脚手架与认证
**Requirements:** FOUND-03
**Status:** Ready

## Goal

实现设置页面：通知频率、语言兴趣、安静时段的配置 UI，数据通过 tauri-plugin-store 持久化。

## Scope

### apps/desktop-ui/src/routes/settings/+page.svelte
1. Settings 页面布局
   - 标题："设置"
   - 分组卡片式布局

2. 通知设置组
   - 标题："通知频率"
   - Radio 选项：实时 / 12h Digest / 24h Digest / 静音
   - 当前值高亮

3. 语言兴趣组
   - 标题："语言兴趣"
   - 自动推断区域（如果有认证 token）：
     - "从你的 Star 仓库推断" 按钮 → 调用 `GET /user/starred` 分析 Top 5 语言
   - 手动选择区域：常见语言复选框网格
     - Rust, TypeScript, JavaScript, Python, Go, Java, C++, C#, Swift, Kotlin, Ruby, PHP, Dart, Shell
   - 已选语言高亮显示

4. 安静时段组
   - 标题："安静时段"
   - 开关：启用/禁用
   - 启用后显示：开始时间 + 结束时间选择器（<input type="time">）
   - 默认：禁用

5. 保存行为
   - 每个设置变更自动保存（即时调用 updateSettings）
   - 保存成功显示短暂 toast "已保存"
   - 保存失败显示错误 toast

### apps/desktop-ui/src/lib/components/SettingsGroup.svelte
- 可复用的设置分组卡片组件
- Props: title, children

### apps/desktop-ui/src/lib/components/LanguagePicker.svelte
- 语言选择器组件
- Props: available, selected, onChange

### Key Design Decisions
- 自动保存（无需"保存"按钮）— 桌面应用常见模式
- 语言推断作为增强功能（需要 token，可能失败则回退到手动）
- 安静时段使用 HTML time input（跨平台一致）
- 设置通过 tauri-plugin-store 存储为 JSON

## Verification

- [ ] Settings 页面可从 Sidebar 底部齿轮进入
- [ ] 通知频率选择后自动保存
- [ ] 语言兴趣复选框可勾选/取消
- [ ] 安静时段开关可切换
- [ ] 安静时段启用后时间选择器出现
- [ ] 所有设置在刷新后保持
- [ ] 设置存储为 JSON 文件（可在应用数据目录查看）
