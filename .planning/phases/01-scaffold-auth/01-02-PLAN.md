# Plan 01-02: GitHub 认证 Tauri 命令

**Phase:** 1 — 项目脚手架与认证
**Requirements:** FOUND-01, FOUND-02
**Status:** Ready

## Goal

实现 GitHub PAT 认证的核心 Tauri 命令：token 验证、安全存储、用户信息获取、仓库基本信息查询。

## Scope

### crates/github_adapter/src/
1. `auth.rs` — GitHub 认证客户端
   - 使用 octocrab 创建认证 Octocrab 实例
   - `validate_token(token: &str) -> Result<User, AuthError>`: 调用 `GET /user` 验证
   - `get_authenticated_client(token: &str) -> Octocrab`: 创建带 token 的客户端
   - `fetch_repo_info(client: &Octocrab, owner: &str, repo: &str) -> Result<RepoBasicInfo, Error>`: 调用 `GET /repos/{owner}/{repo}`

2. `lib.rs` — 模块导出

### crates/runtime_tauri/src/
1. `commands/auth.rs` — 认证 Tauri 命令
   - `#[tauri::command] async fn validate_github_token(token: String) -> Result<ValidateTokenResponse, String>`
     - 调用 github_adapter::validate_token
     - 成功时返回 UserDto
     - 失败时返回具体错误信息
   - `#[tauri::command] async fn store_github_token(app: AppHandle, token: String) -> Result<(), String>`
     - 使用 keyring 存储 token（service: "geek-taste", user: "github-pat"）
   - `#[tauri::command] async fn load_github_token() -> Result<String, String>`
     - 从 keyring 加载 token
   - `#[tauri::command] async fn remove_github_token() -> Result<(), String>`
     - 从 keyring 删除 token
   - `#[tauri::command] async fn get_current_user() -> Result<Option<UserDto>, String>`
     - 加载 token → 创建 client → 调用 /user → 返回 UserDto
   - `#[tauri::command] async fn fetch_repo_info(owner: String, repo: String) -> Result<RepoBasicInfo, String>`
     - 需要已认证的 client

2. `commands/mod.rs` — 模块导出

3. `commands/settings.rs` — 设置 Tauri 命令
   - `#[tauri::command] async fn get_settings(app: AppHandle) -> Result<SettingsDto, String>`
     - 从 tauri-plugin-store 读取 settings.json
   - `#[tauri::command] async fn update_settings(app: AppHandle, settings: UpdateSettingsRequest) -> Result<SettingsDto, String>`
     - 写入 tauri-plugin-store

4. `lib.rs` — 注册所有 commands

### Tauri Plugin: keyring 集成
- 在 `src-tauri/Cargo.toml` 添加 `keyring` 依赖（已在 workspace 配置）
- 在 `src-tauri/src/lib.rs` 不需要额外插件注册（keyring 是直接调用）

### Key Design Decisions
- keyring service name: "geek-taste", key: "github-pat"
- 认证状态通过 keyring 中是否有有效 token 判断
- octocrab 客户端每次需要时重新创建（不持久化实例）
- 错误信息直接从 GitHub API 响应解析
