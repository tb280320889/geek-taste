# Plan 01-06: Home 欢迎页与仓库探索

**Phase:** 1 — 项目脚手架与认证
**Requirements:** FOUND-02
**Status:** Ready

## Goal

Home 页面显示用户欢迎信息；TopK 页面提供仓库探索输入框，可查询任意仓库的基本信息。

## Scope

### apps/desktop-ui/src/routes/+page.svelte (Home)
1. Home 页面内容
   - 欢迎卡片：用户头像 + "欢迎回来，{name}" 
   - 快捷入口网格（占位，后续 Phase 填充）：
     - "发现趋势" → /topk
     - "管理订阅" → /subscriptions  
     - "Agent 资源" → /resources
   - 未认证时显示"请先连接 GitHub"提示卡（由 layout 处理）

### apps/desktop-ui/src/routes/topk/+page.svelte (TopK)
1. 仓库探索区域
   - 输入框：placeholder "输入仓库名称，如 facebook/react"
   - 支持格式：`owner/repo` 或完整 GitHub URL
   - 搜索按钮或 Enter 触发

2. 仓库详情 Modal
   - 弹出式 Modal，半透明遮罩
   - 内容：
     - 仓库名 (full_name) + 外链图标（打开 GitHub 页面）
     - Description
     - Stars ⭐ + Forks 🔀 数字
     - Language（彩色标签）
     - Topics（标签组）
   - 关闭按钮 + 点击遮罩关闭

3. 错误处理
   - 仓库不存在：显示"未找到该仓库"
   - 网络错误：显示"无法连接 GitHub"
   - 未认证：显示"请先连接 GitHub"

### apps/desktop-ui/src/lib/components/RepoInfoModal.svelte
- 仓库详情 Modal 组件
- Props: repoInfo (RepoBasicInfo | null), onClose
- 动画：淡入 + 居中缩放

### Key Design Decisions
- Home 页面 v1 为简单欢迎 + 快捷入口（Phase 3 后填充信号摘要）
- 仓库探索放在 TopK 页面顶部（Phase 2 后添加排名列表）
- Modal 使用 CSS animation（无额外动画库）
- 输入框支持 owner/repo 和完整 URL 两种格式
