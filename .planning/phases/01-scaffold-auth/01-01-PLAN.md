# Plan 01-01: 领域模型与共享契约

**Phase:** 1 — 项目脚手架与认证
**Requirements:** FOUND-01, FOUND-03
**Status:** Ready

## Goal

创建认证和设置相关的领域模型、DTO 类型定义，为 Tauri 命令和前端提供类型安全的契约层。

## Scope

### crates/domain/src/
1. `auth.rs` — AuthToken, User 领域对象
   - `AuthToken` struct: token (String, masked on Debug), created_at, last_validated
   - `User` struct: login, name, avatar_url, html_url
   - `AuthError` enum: InvalidToken, RateLimited, NetworkError, Unknown
   - impl: `AuthToken::new(token)`, `AuthToken::is_expired()` (v1 简化: 启动时验证)
   - impl: `User::from_github_response()` (从 octocrab response 转换)

2. `settings.rs` — Settings 领域对象
   - `Settings` struct: notification_frequency (enum: Realtime/Digest12h/Digest24h), language_interests (Vec<String>), quiet_hours (Option<QuietHours>)
   - `QuietHours` struct: start (NaiveTime), end (NaiveTime)
   - `NotificationFrequency` enum: Realtime, Digest12h, Digest24h, Muted
   - impl: `Settings::default()` — 默认 Digest12h, 空语言, 无安静时段

3. `lib.rs` — 模块导出

### crates/shared_contracts/src/
1. `auth_dto.rs` — 认证相关的请求/响应类型
   - `AuthStatus` enum: Authenticated(UserDto), Unauthenticated, TokenExpired
   - `UserDto` struct: login, name, avatar_url (serde, TS export)
   - `ValidateTokenRequest` struct: token (String)
   - `ValidateTokenResponse` struct: success, user (Option<UserDto>), error (Option<String>)

2. `settings_dto.rs` — 设置相关的请求/响应类型
   - `SettingsDto` struct: notification_frequency, language_interests, quiet_hours (serde, TS export)
   - `QuietHoursDto` struct: start ("HH:MM"), end ("HH:MM")
   - `UpdateSettingsRequest` struct: 包含所有可更新字段

3. `repo_dto.rs` — 仓库基本信息 DTO
   - `RepoBasicInfo` struct: full_name, description, stargazers_count, forks_count, language, topics, html_url (serde, TS export)

4. `lib.rs` — 模块导出 + `#[cfg_attr(feature = "ts", derive(ts_rs::Ts))]` 如果使用 ts-rs

### Key Design Decisions
- 领域模型不依赖 Tauri/Svelte — 纯 Rust struct
- DTO 用 serde 序列化，前端通过 Tauri IPC 调用
- AuthToken 在 Debug 时 mask token 值（安全考虑）
- Settings 使用 JSON 文件存储（tauri-plugin-store），不需要 SQLite
